Site Report

Introduction

Hi!,I am Purnika karki.In this page, I will be writing about my experience in this CSY1063 Web Development Module and the portfolio website that I have created.This portfolio consist of five different pages and uses various HTML and CSS concepts.I have tried my level best to showcase what I have learned, the challenges I faced and how I evolved throughout this module whilebuilding the website from scratch.

Module Overview

At the start of this module, I was curious as well as excited to learn about web development. Since I had studied Computer Science in +2, I had basic knowledge about HTML, but CSS was completely new to me. Throughout the module, I was fascinated to learn about different properties of HTML and CSS. Learning concept such as floats, flexbox and grid was a bit confusing at first, and I struggled particularly with flexbox because I was already used to float properties ,which was taught at the beginning of the module. However, through continuous practice and experimenting layouts, I gradually understand these concepts.CSS properties such as background, color, transistion, animation were really fascinating to learn and I enjoyed alot while padding, border, margin, transform are still a bit challenging for me. Despite this, I can clearly see my progress and how much I have learned throughout this module. I would like to thank our module teacher, Mr. Ankit Thapa, who cleared all our queries by explaining the properties during lectures and then implementing them in tutorial classes, which helped make my concepts much clearer.

Design Challenges and learning

During development, I faced several challenges in both HTML and CSS. Initially. I used section elements without proper headings, which affected structure and accessibility. I also used multiple div elements without managing spacing properly, which made layout fell cramped and unbalanced. Realising this later in the development process, I revisited and improved spacing and organization. For styling I initially used Flexbox for my layout design. However,after reviewing the overall appearance, I felt that Flexbox limited my layout control for more complex sections.As a result, I decided to switch to CSS grid. This required rewriting a large portion of my CSS, but it significantly improved layout alignment and consistency across pages. Implementing a responsive navigation without JavaScript was one of the hardest parts, so I used a checkbox-based hamburger menu controlled purely with CSS. Designing for mobile view was difficult, particularly deciding appropriate max-width values, and the index page was especially challenging as it serves as the first impression of my portfolio.

Design Decision

For the Layout, I chose CSS Grid because it gave me better control over complex sections and helped maintain consistency accross pages. To plan the pages I brainstromed and started with making wireframes for each to organise content and visual flow. I even watched tutorials on Youtube browsed website online for layout ideas. I decided on dark background theme with blue and light accents to make the content visually clear, while adding subtle pink shadow to enhance depth and highlight important elements.I wanted hamburger icon on mobile page so for specific concept like toggle menu for mobile view and for project.html readmore button, I referred to external sources, including video tutorial, AI guidance, to understand hoe they work, but implemented them using my own design approach. I also experimented different fonts and finally chosee Poppins with sans-seriff because they are modern, readable and fit my overall aesthetics, and tested spacing and layout to ensure the pages were balances and responsive.

The portfolio.css file is shared external stylesheet used accross all pages of the portfolio to maintain consistent design. It controls the overall layout, color, scheme, typography, navigation styling and responsiveness for both desktop and mobile views.

During CSS validation, a warning was shown as I have used background-clip: text property to create a gradient text effect for my logo.Although this property is marked as deprecated by the W3C validator, it is widely supported by modern browsers. A fallback solid text colour has also been defined to ensure the logo remains visible and readable if the gradient effect is not supported. This warning does not affect the functionality, accessibility, or responsiveness of the website.

Overall, this module was a valuable learning experience that helped me understand the fundamentals of web development.
